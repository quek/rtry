# レポートテンプレート

調査結果は以下の形式で日本語でまとめる。

```markdown
## 調査結果: [機能名]

### 各プロジェクトの実装

#### tsf-tutcode
- ファイル: ...
- パターン: ...
- API 呼び出し: ...

#### CorvusSKK / SampleIME
- ...

### API リファレンス・ガイドからの知見
- 公式ドキュメントで確認した仕様・制約
- 使用上の注意点やベストプラクティス

### windows crate の API シグネチャ
- 確認した関数とそのシグネチャ（C++ との差異）

### 推奨アプローチ
- rtry での実装方針
- 採用する設計パターンとその理由

### CUAS/互換性の注意点
- CUAS 環境（Emacs 等 IMM32 アプリ）での制限と対策
- エッジケースの扱い

### 参考コード
- 具体的なコード例（C++ → Rust への変換ポイント）
```
